Section 1 — Usage Guide

🔑 Logging In & Out

    1
    Click "Admin Login"

    At the bottom of the left sidebar.

    2
    Enter the password

    The admin password is stored as a Netlify environment variable (ADMIN_PASSWORD). Contact the developer to change it.

    3
    Admin Mode activates

    A green "Admin Mode" badge appears under the NPS logo, a "Logout" button replaces the login button, and extra controls appear throughout the portal.

⚠ Admin tokens are stored in the browser. Always log out when using a shared device.

📢 Posting & Editing Announcements

Admin controls appear at the top of the Announcements tab.

  • Title & Content — the main text of the announcement.
  • Type — sets the colour stripe on the card. Options: General, Academic, Inter-School, MUN, Inter-House, Sports.
  • Link Calendar Events — tick one or more events from the list to show them as chips on the card. Events must be added to the calendar first.
  • Attachment — upload a PDF, image, Word doc, or PowerPoint. It is uploaded to the NPS Announcement Attachments Drive folder automatically and a "View Attachment" link appears on the card.
ℹ To edit or delete a posted announcement, scroll to the card and click the Edit or Delete button that appears at the bottom of each card when you are in Admin Mode.

📅 Managing Calendar Events

  • Adding — use the "Add Calendar Event" form in the Announcements tab admin panel. Set the date, name, and type (Academic / Cultural / Sports / Holiday).
  • Navigating months — use the ← → arrows on either side of the calendar title to move between months.
  • Editing / deleting — in Admin Mode, click any event chip inside the calendar grid. An edit modal opens with pre-filled fields and a Delete button.
⚠ Calendar events cannot be deleted while they are linked to announcements. Re-edit those announcements to uncheck the event first.

💬 Managing Suggestions

Suggestions are shown in three columns: Submitted → Under Review → Resolved.

  • View details — click any suggestion card to open its full detail view (visible to all users).
  • Move status — inside the detail modal, use the "Move to Under Review ▶" / "Back to Submitted ◀" etc. buttons. The card moves columns immediately.
  • Edit — scroll down in the detail modal to the "Edit Suggestion" form. You can change the category, title, and description.
  • Delete — click "✕ Delete" in the detail modal. This permanently removes it from the spreadsheet.

🏆 House Points

The House Points tab shows current totals as summary cards and a full history ledger.

  • Adding an entry — scroll to the bottom of the House Points tab. Fill in a Reason, then enter a point change (positive or negative) for one or more houses. Submitting adds one ledger row per house that had a value.
  • Negative points — just enter a negative number (e.g. -10). It appears in red in the ledger.
  • History is public — all users see the full ledger and totals. There is no way to hide rows; think before you post a reason.
⚠ There is currently no delete/edit for house point entries. If you make a mistake, add a correcting entry with an explanation in the Reason field (e.g. "Correction — reversed duplicate entry").

📄 Past Papers / Submissions

Students upload files via the Submission Drop-Box on the Past Papers tab.

Files are named automatically:

{year}_Grade{grade}_{subject}_{examType}_{suffix}
Example: 2024-2025_Grade11_Physics_Half-Yearly_scan1.pdf

Files first land in the PENDING_DRIVE_FOLDER_ID folder in Google Drive (set in Netlify env vars).

  • Reviewing — in Admin Mode, a "Submitted Documents" table appears at the bottom of the Past Papers tab. Click "View File" to open the PDF in Drive.
  • Approving — edit the suffix in the table if needed, then click "Approve PDF". The site renames the file and moves it to the appropriate grade folder automatically.
  • Naming tip — while pending, the suffix defaults to the student's original file name. Reviewers should replace it with a clean suffix before approval.
Section 2 — Editing Category Dropdowns

🗂 Overview

Dropdowns are defined in two places: the HTML (what users see) and sometimes the JavaScript (colour maps, logic). A third place — the spreadsheet header row — is for your own reference and doesn't need to match exactly, but keeping it consistent is good practice.

All files are in the root of the project. Edit them in any text editor or VS Code.

📣 Announcement Types

Where to edit: index.html and main.js

1. Add/remove options in index.html — find the "Post Announcement" form:

<select id="admin-ann-type">
    <option value="General">General</option>
    <option value="Academic">Academic</option>
    <option value="Inter-School">Inter-School</option>
    <option value="MUN">MUN</option>
    <option value="Inter-House">Inter-House</option>
    <option value="Sports">Sports</option>
</select>

2. Add a colour in main.js — find the ANN_TYPE_COLORS object near the top:

const ANN_TYPE_COLORS = {
    General:        '#64748b',
    Academic:       '#f59e0b',
    'Inter-School': '#3b82f6',
    MUN:            '#8b5cf6',
    'Inter-House':  '#f97316',
    Sports:         '#10b981',
    // Add new type here, e.g.:
    // 'Cultural':  '#ec4899',
};

The key must match the value attribute in the HTML exactly (case-sensitive).

ℹ Also update the type list in the Edit Announcement modal inside openAnnouncementEdit() in main.js — look for the line const annTypes = [...].

💬 Suggestion Categories

Where to edit: index.html and main.js

1. Add/remove options in index.html — find the "Submit a Suggestion" form and the "Edit Suggestion" form:

<select id="sugg-category">
    <option value="Infrastructure">Infrastructure</option>
    <option value="Academics">Academics</option>
    <option value="Culture">Culture</option>
    <option value="Sports">Sports</option>
    <option value="General">General</option>
</select>

2. Add a colour in main.js — find the CATEGORY_COLORS object:

const CATEGORY_COLORS = {
    Infrastructure: { bg: 'rgba(249,115,22,0.15)', border: '#f97316', text: '#fb923c' },
    Academics:      { bg: 'rgba(59,130,246,0.15)',  border: '#3b82f6', text: '#60a5fa' },
    Culture:        { bg: 'rgba(236,72,153,0.15)',  border: '#ec4899', text: '#f472b6' },
    Sports:         { bg: 'rgba(16,185,129,0.15)',  border: '#10b981', text: '#34d399' },
    General:        { bg: 'rgba(139,92,246,0.15)',  border: '#8b5cf6', text: '#a78bfa' },
};

Each entry needs three values: bg (card background), border (left accent), and text (badge colour).

Also update the edit-suggestion dropdown in openSuggestionDetail() in main.js — look for the array ['Infrastructure','Academics','Culture','Sports','General'].

📅 Calendar Event Types

Where to edit: index.html and style.css

1. Add/remove options in index.html — there are two selects, one in "Add Calendar Event" and one in the edit modal template inside openCalendarEventEdit() in main.js:

<select id="admin-cal-type">
    <option value="Academic">Academic</option>
    <option value="Cultural">Cultural</option>
    <option value="Sports">Sports</option>
    <option value="Holiday">Holiday</option>
</select>

2. Add colours in style.css — find the event type rules near the bottom:

.event-Academic { border-left-color: #f59e0b; background: rgba(245,158,11,0.2); }
.event-Cultural { border-left-color: #ec4899; background: rgba(236,72,153,0.2); }
.event-Sports   { border-left-color: #10b981; background: rgba(16,185,129,0.2); }
.event-Holiday  { border-left-color: #ef4444; background: rgba(239,68,68,0.2); }
/* Add: .event-NewType { border-left-color: #???; background: rgba(?,?,?,0.2); } */

The CSS class name must be .event-{TypeValue} where TypeValue exactly matches the option value.
